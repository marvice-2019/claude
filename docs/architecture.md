## Data Flow Summary

1. **Call arrives** вҶ’ Twilio/Exotel sends webhook to n8n
2. **Audio streamed** вҶ’ Deepgram converts speech to text in real-time
3. **Language detected** вҶ’ First 2 utterances determine language, stored in session
4. **AI processes** вҶ’ Claude analyzes intent, emotion, context; generates response
5. **Humanization** вҶ’ Response passes through humanization engine (delays, fillers, variation)
6. **TTS converts** вҶ’ ElevenLabs/Azure converts text to natural speech
7. **Audio returned** вҶ’ Streamed back to caller via Twilio/Exotel
8. **Loop continues** вҶ’ Steps 2-7 repeat for each turn until call ends
9. **Post-call** вҶ’ Data synced to Sheets, WhatsApp confirmation sent, CRM updated, logs stored

## Key Design Principles

- **Sub-1-second latency** target for response generation
- **Stateful conversations** вҖ” every turn has full context of prior turns
- **Graceful degradation** вҖ” if Deepgram fails, fall back to Whisper; if ElevenLabs fails, fall back to Azure
- **No dead air** вҖ” filler phrases ("One moment pleaseвҖҰ") stream immediately while AI processes
- **Human escalation** вҖ” any point the caller says "talk to a person" or agent detects frustration > threshold
